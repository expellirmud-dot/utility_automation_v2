# Certification Governance

## Requirements

- Preserve deterministic governance.
- Preserve certification integrity.
- Report exact validation evidence.
- Never weaken tests or certifier checks to make a task pass.
- If certification policy must change, make it explicit, minimal, path-bound, token-bound, and controller-approved.

## Required Evidence

Report:

1. exact command
2. raw output
3. pass/fail result
4. affected checks
5. remaining risk
